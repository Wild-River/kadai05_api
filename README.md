# ①課題名

おでかけBGM

## ②課題内容（どんな作品か）

- 写真をアップロードすると、その雰囲気に合った音楽（BGM）を自動生成して再生できるアプリです。<br>
  写真の色合いや時間帯、被写体、感情などをAIが読み取り、それに合う音楽を作曲します。

## ③アプリのデプロイURL

https://photo-music-2026gs.web.app/

## ④アプリのログイン用IDまたはPassword（ある場合）

- ログインなし（写真を選んでボタンを押すだけで利用できます）

## ⑤工夫した点・こだわった点

- 2つのAIを組み合わせた二段構成にしました。まず Claude（vision）が写真を解析して「どんな音楽が合うか」のプロンプトを生成し、そのプロンプトを ElevenLabs（Music API）に渡して実際の音楽を生成しています。
- APIキー（Claude / ElevenLabs）は、フロントに絶対に出さないよう Firebase Functions のシークレットに格納し、サーバー側（Cloud Functions）でのみ使用しています。
- 処理の流れを「写真 → Claude → ElevenLabs → Storage → 再生URL」と1つの関数（onCall）にまとめ、各段階でエラーを切り分けられるようにしました。
- 写真は送信前にブラウザ側でリサイズ（canvasで長辺1500pxに縮小）してから送ることで、通信量を抑えています。
- 生成した音楽は Firebase Storage に保存し、その再生URLを返す仕組みにしました（音声ファイル本体とURLを分けて扱う設計）。

## ⑥難しかった点・次回トライしたいこと（又は機能）

- フロント（Vite）とサーバー（Cloud Functions）を1つのプロジェクト内に分けて構成するのが初めてで、フォルダ構成や設定ファイルの置き場所に苦労しました。
- 新しいFirebaseプロジェクトで初めて関数をデプロイする際、ビルド用サービスアカウントの権限不足や、Cloud Run の呼び出し権限（公開アクセス）の設定など、Google Cloud側の細かい設定でつまずきました。ログを読んで原因を一つずつ切り分けて解決しました。
- Claude・ElevenLabs・Firebase Storage の3つを連携させる際、どこで失敗しているかをログから特定する作業が大変でしたが、良い経験になりました。
- 次回は、生成した曲を一覧として保存できる「マイBGMライブラリ」や、SNSでシェアできる機能を追加してみたいです。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）

「写真の雰囲気から音楽が作られたら面白いのでは」という思いつきから制作しました。<br>
2つのAI（Claudeで解析、ElevenLabsで作曲）を組み合わせる構成や、Firebase Functions を使ったAPIキーの安全な管理など、初めての挑戦が多い課題でしたが、実際に写真から音楽が生成されたときは感動しました！
